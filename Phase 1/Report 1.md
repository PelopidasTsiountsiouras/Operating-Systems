# 🧮 TinyShell — Μικρό Unix-like Κέλυφος (Phase 1)

## 📘 Περιγραφή

Το **TinyShell** είναι ένα εκπαιδευτικό, ελαφρύ Unix-like κέλυφος γραμμένο σε **C**, σχεδιασμένο για Linux.  
Υλοποιήθηκε στο πλαίσιο του μαθήματος **Λειτουργικά Συστήματα** με στόχο την κατανόηση της λειτουργίας των POSIX system calls `fork()`, `execve()`, και `waitpid()` — των βασικών μηχανισμών που χρησιμοποιούν τα λειτουργικά συστήματα για τη δημιουργία και διαχείριση διεργασιών.

Το έργο αποτελεί την **πρώτη φάση** της ανάπτυξης ενός πληρέστερου shell, επικεντρωμένης στην εκτέλεση βασικών εντολών και την αλληλεπίδραση με τον χρήστη.

---

## 🎯 Στόχοι φάσης 1

Το TinyShell πρέπει να μπορεί να:

1. Εμφανίζει ένα **prompt** (`tinyshell$`).
2. Αναλύει τα **ορίσματα** μιας εντολής.
3. Εντοπίζει εκτελέσιμα στο **PATH**.
4. Εκτελεί προγράμματα μέσω `fork()` και `execve()`.
5. Αναφέρει τους **κωδικούς εξόδου** (exit codes).
6. Τερματίζει ομαλά με την εντολή `exit` ή με EOF (`Ctrl+D`).

## Οδηγίες Εκτέλεσης

1. Μεταγλώτισση: 
    ```
    make
    ```

2. Εκτέλεση:
    ```
    ./tinyshell
    ```

3. Τερματισμός:
    - Με την εντολή `exit`
    - Ή με `Ctrl + D`

### Παράδειγμα Εκτέλεσης
```
tinyshell$ ls
Makefile  tinyshell  tinyshell.c
[pid 19706] exited with code 0

tinyshell$ ls -l /
drwxr-xr-x  2 root root 4096 ...
[pid 19800] exited with code 0

tinyshell$ /bin/echo hello world
hello world
[pid 19862] exited with code 0

tinyshell$ cat /etc/hostname
user-HP-Laptop
[pid 19953] exited with code 0

tinyshell$ foo
tinyshell: command not found: foo

tinyshell$ exit 7
Exiting (exit 7)
```

## Τεχνική Ανάλυση


| Συνάρτηση   | Περιγραφή                                |
| ----------- | ---------------------------------------- |
| `fork()`    | Δημιουργεί παιδί διεργασίας              |
| `execve()`  | Εκτελεί νέο πρόγραμμα στο παιδί          |
| `waitpid()` | Αναμένει την ολοκλήρωση της διεργασίας   |
| `access()`  | Ελέγχει αν το αρχείο είναι εκτελέσιμο    |
| `getenv()`  | Διαβάζει τη μεταβλητή περιβάλλοντος PATH |

Η ροή εκτέλεσης είναι:
1. Εμφάνιση prompt
2. Ανάγνωση εντολής
3. Parsing (split)
4. Έλεγχος αν είναι `exit`
5. Αναζήτηση εκτελέσιμου στο PATH
6. Δημιουργία παιδιού (fork)
7. Εκτέλεση (execve)
8. Αναμονή (waitpid)
9. Εμφάνιση αποτελέσματος

---

## Συμπεράσματα

Το TinyShell επιτυγχάνει πλήρως τους στόχους της πρώτης φάσης:
- Υλοποιεί βασικό command loop.
- Εκτελεί προγράμματα μέσω POSIX system calls.
- Διαχειρίζεται σωστά τα exit codes.
- Παρέχει φιλικό prompt και ομαλό τερματισμό.
